三、各模块功能需求与整体设计目标3.1 高级数学运算模块（指数、自然对数）1）功能需求原始计算器仅支持加减乘除四则基础运算，无法满足高等数学基础计算需求，因此新增幂运算与自然对数两类常用高级运算；运算逻辑需要和原有四则运算解耦，不破坏原有稳定代码，同时增加非法输入拦截机制，防止程序闪退、计算报错。2）细分功能细则
幂次指数运算 \(x^n\)：支持用户分别输入底数、指数，支持小数底数、正负整数指数；单独增加输入弹窗，分开采集两个参数；
自然对数 \(\ln(x)\)：仅接收大于 0 的自变量，若输入小于等于 0 的数字，弹窗提示非法输入，直接终止计算流程；
运算结果统一精度处理：所有高级运算输出结果统一保留 6 位小数，解决浮点数连续运算精度丢失问题。
3）整体设计目标将全部数学运算逻辑抽离独立工具类，主页面仅负责接收用户点击事件、弹窗交互、结果回显；实现 UI 交互与数学计算逻辑分层解耦，后续可快速新增其他运算，代码复用性更强。3.2 计算历史持久化存储模块1）功能需求原始计算器所有运算数据仅保存在页面内存中，应用退出、页面销毁后所有记录直接丢失；新增本地持久化存储，永久保存每一次完整运算的表达式、计算结果、运算时间，提供独立页面查看、复用、清空记录。2）细分功能细则
自动存储：每次点击等于按钮运算成功后，自动封装数据写入本地 Preferences 持久化容器，无需手动保存；
历史列表页面：独立路由页面展示全部历史条目，每条记录显示完整表达式与对应结果；
记录复用：点击任意一条历史记录，自动将表达式回填至计算器首页输入面板，可直接二次运算；
一键重置功能：页面底部增加清空按钮，点击后删除本地全部存储记录，无残留缓存；
3）整体设计目标单独封装一套存储读写工具类，统一管理本地数据增、查、删操作；历史数据封装实体类统一规范数据结构，页面仅调用工具方法，不直接操作底层存储 API，降低代码耦合度。3.3 白天 / 夜间全局主题切换模块1）功能需求原始工程仅有一套浅色界面，夜间使用刺眼；新增双套全局主题，一键切换浅色、深色模式，切换后永久保存用户选择，重启应用自动读取上次使用的主题。2）细分功能细则
全局切换开关：计算器首页顶部放置主题切换组件，点击即可实时切换整套界面色彩；
全页面色彩适配：计算器首页、历史记录页面、函数绘图页面全部 UI 组件同步切换配色；覆盖页面背景、按钮底色、按钮文字、输入框边框、画布背景、提示文字；
状态持久记忆：用户切换主题后，选择永久存入本地，关闭应用再次打开无需重复切换；
3）整体设计目标使用应用级全局响应式状态管理主题变量，所有页面 UI 组件绑定全局主题色，切换时无需页面重载，实时无卡顿刷新全部界面样式；区分两套完整配色常量，统一在资源文件中管理，方便后期修改配色。3.4 初等函数图像绘图模块（7 种指定函数）1）功能需求在计算器基础计算能力之上拓展可视化绘图功能，内置 7 种固定初等函数，无需用户手动输入表达式，直接点击函数按钮即可自动生成对应平面曲线图像，配套坐标系画布，支持拖拽平移查看图像细节，针对每一种函数定义域单独做异常渲染拦截。2）支持绘制的固定函数列表\(\sin x、\cos x、\displaystyle\frac{1}{x}、x^2、x^3、e^x、\ln x\)3）细分功能细则
独立绘图页面：单独路由页面承载绘图功能，与计算器计算页面完全隔离，互不占用页面空间；
函数选择控件：页面内横向排列 7 个函数选择按钮，点击任意按钮立即渲染对应函数图像；
坐标系自动适配：画布内置标准平面直角坐标系、X/Y 刻度网格，程序自动根据函数值域、定义域自适应画布显示区间；
定义域渲染拦截（核心容错逻辑）：

\(\displaystyle\frac{1}{x}\)：x=0 处存在断点，渲染时跳过 x=0 坐标，不绘制该点，避免曲线异常断裂；
\(\ln x\)：定义域仅 x>0，x≤0 区间不生成任何坐标点，仅在 x>0 区域绘制曲线；
\(\sin x、\cos x、x^2、x^3、e^x\)：全体实数域均可正常渲染，无区间限制；


画布拖拽交互：鼠标拖动画布可平移坐标系，查看图像局部细节；
4）整体设计目标抽离独立绘图工具类，统一完成坐标采样、函数值计算、线条渲染逻辑；页面仅负责按钮点击事件、画布容器渲染，绘图算法与 UI 页面分层，降低页面代码冗余。四、新增工程文件与目录结构完整变更说明4.1 新增完整目录树（仅展示新增 / 修改部分）plaintextentry/src/main/ets

五、四大模块核心代码实现详细拆解5.1 高级数学运算模块（MathCalculateUtil.ets 工具类详解）工具类封装全部指数、自然对数底层计算逻辑，对外仅暴露简洁调用方法，首页页面无需关心底层计算细节。1）幂次指数运算方法实现
入参：接收两个浮点参数 base（底数）、exp（指数）；
底层计算：调用系统内置 Math.pow (base, exp) 完成幂运算；
边界异常判断：

若底数为 0 且指数小于 0，数学无意义，返回错误标识；
计算完成后统一调用精度处理函数，保留 6 位小数；


页面交互联动逻辑：
Index.ets 页面新增xⁿ按钮，用户点击后弹出双层输入弹窗，分别输入底数、指数；弹窗确认按钮触发调用 MathCalculateUtil 幂运算方法；若返回错误标识，页面弹出 Toast 提示非法输入；计算正常则将结果回填至计算器输入显示面板。
2）自然对数 ln (x) 运算方法实现
入参：浮点参数 x（对数自变量）；
前置定义域校验：函数执行第一行判断x <= 0，直接返回错误状态；
正常计算逻辑：使用系统 Math.log (x) 实现自然对数运算；
精度统一处理：计算结果强制保留 6 位小数，统一界面显示格式；
页面交互联动：
首页新增ln按钮，点击弹出单输入框弹窗，输入数字后确认；工具类校验不通过则弹窗提示 “自变量必须大于 0”，终止计算；校验通过则将 ln 运算结果回显至计算器主面板。
5.2 计算历史持久化模块（StorageUtil + HistoryModel + HistoryPage）1）数据实体类 HistoryModel.ets统一规范单条历史记录的数据结构，每一条记录固定包含 4 个字段：
expression：用户完整输入运算表达式字符串；
result：该表达式最终计算结果；
timeStamp：运算完成时间戳，用于区分记录先后顺序；
id：唯一标识每条记录，便于删除、匹配复用；
所有历史数据以 HistoryModel 数组格式统一存储、读取，数据结构标准化，避免存储字段混乱。
2）持久化工具类 StorageUtil.ets 三大核心方法
saveHistory (item: HistoryModel) 自动存储方法
每次计算器运算成功后自动调用，接收封装完成的 HistoryModel 对象，通过 Preferences 容器将数组持久写入本地文件；多次运算自动追加至历史数组末尾，不会覆盖原有记录。
getAllHistory(): Array<HistoryModel> 读取全部记录方法
历史页面加载时调用，读取本地完整存储的历史数组并返回，页面通过 ForEach 循环批量渲染全部记录条目。
clearAllHistory () 一键清空方法
删除 Preferences 内全部历史存储数组，本地无残留数据，调用后刷新历史页面列表，列表清空。
3）页面完整联动流程
计算器首页 Index.ets：点击等于运算成功，自动组装 HistoryModel 数据，调用 StorageUtil.saveHistory 完成本地保存；页面顶部新增「历史」按钮，点击使用路由 API 跳转 HistoryPage；
历史页面 HistoryPage.ets：页面生命周期 onAboutToAppear 时调用 getAllHistory 读取本地数据；通过 ForEach 循环渲染多个 HistoryItem 复用组件；
HistoryItem 单条记录组件：每条记录展示表达式 + 结果，点击条目自动提取 expression 表达式字符串，路由回传至计算器首页，回填至输入面板直接复用计算；页面底部绑定清空按钮，点击执行 clearAllHistory 清空全部本地记录。
5.3 白天 / 夜间全局主题切换模块（GlobalTheme + ThemeSwitch 组件）1）全局主题状态模型 GlobalTheme.ets
使用 @AppStorage 应用级全局持久化变量 isDarkMode：布尔值，true = 夜间深色模式，false = 白天浅色模式；变量存储于应用全局，所有页面共享同一套状态，切换后自动写入本地，重启应用自动读取；
导出两套完整色彩对象 lightColor、darkColor：两套对象包含页面背景、按钮背景、文字、边框、绘图画布、提示文字全部色值；页面任意 UI 组件直接读取全局主题颜色对象，无需单独判断模式。
2）复用切换组件 ThemeSwitch.ets独立封装开关 UI 组件，仅提供点击交互逻辑；点击开关时翻转 isDarkMode 布尔值；全局变量变化后，所有页面绑定色彩的组件自动响应刷新，无需页面重载，实时切换整套界面配色。3）全页面色彩绑定联动逻辑计算器首页 Index、历史页面 HistoryPage、绘图页面 DrawPage 中所有 Text、Button、容器、画布背景均绑定 GlobalTheme 导出的动态颜色；切换开关后，三个页面色彩同步变更，无页面割裂、样式不同步问题。5.4 7 种固定函数绘图模块（DrawPage + DrawUtil 绘图工具类）1）绘图页面 DrawPage 交互逻辑页面横向排列 7 个独立按钮，分别对应 \(\sin x、\cos x、1/x、x^2、x^3、e^x、\ln x\)；点击任意函数按钮，标记当前选中函数标识，自动调用 DrawUtil 工具类重新渲染画布曲线；页面内置画布容器，支持拖拽平移坐标系，查看图像局部。2）绘图工具类 DrawUtil 底层渲染逻辑
坐标系基础绘制：先渲染 X 轴、Y 轴、均匀刻度网格，画布中心为坐标原点；
分段采样逻辑：在画布可视区间均匀生成大量 x 坐标采样点，根据选中的函数标识代入对应公式计算对应 y 值；将连续采样点连线，生成平滑函数曲线；
分函数定义域拦截处理（关键容错逻辑）

\(\displaystyle\frac{1}{x}\)：采样过程跳过 x=0 坐标点，不在断点处生成连线，避免图像异常；
\(\ln x\)：仅对 x>0 区间进行采样计算，x≤0 区间直接跳过，不渲染任何曲线；
\(\sin x、\cos x、x^2、x^3、e^x\)：全体实数区间正常采样渲染，无区间截断；


画布自适应逻辑：工具类自动读取当前选中函数的值域范围，动态调整画布显示区间，保证完整曲线全部展示在画布可视范围内。
