三、entry/src 核心源码分层详解
1. src/main/ets 业务逻辑层
EntryAbility.ets：应用入口 Ability，管控应用生命周期（启动、切后台、销毁），计算器程序初始化入口；
pages页面目录：存放计算器主页面Index.ets，页面内实现数字按钮、运算符、结果显示、计算逻辑；
components（如有）：按钮、输入框复用 UI 组件；
utils（如有）：运算逻辑封装工具函数，处理四则运算、小数、异常除零。
2. src/main/resources 模块私有资源
计算器页面专属图片、文字、配色，区分中英文多语言目录，存放按钮图标、界面背景、文字常量。
3. src/main/profile
main_pages.json页面路由配置，注册计算器首页，定义程序默认启动页面。
4. module.json5（entry 模块核心配置）
注册页面路由、声明 Ability 组件、配置窗口尺寸、应用权限，绑定 entry 模块编译与依赖规则。

四、计算器原有功能业务流程
启动应用，EntryAbility初始化，加载module.json5路由跳转计算器首页；
页面加载资源文件渲染数字键、运算符、清除、等于按钮 UI；
点击数字 / 运算符，页面缓存输入表达式并实时展示；
点击等于，调用运算工具函数完成四则运算，展示计算结果；
点击清除，重置输入与结果面板。
五、现有工程优缺点 & 拓展铺垫（为任务 3 新增功能做铺垫）
优点：工程分层规范，计算逻辑与 UI 页面分离，代码简洁易拓展；
不足：无历史计算记录、无正负转换、无小数精度优化、无深色主题切换、数据仅内存存储，退出后记录丢失，后续任务 3 可新增以上拓展功能。

3 项目根目录全部文件 / 文件夹详细解析

3.1 .hvigor 编译缓存目录
该文件夹由 hvigor 编译工具自动生成，存放项目编译全过程临时文件，禁止手动修改、提交。
cache：编译缓存，缓存依赖解析结果、编译中间文件，提升二次编译速度；
dependencyMap：依赖映射表，记录项目所有依赖包路径、版本、依赖关联关系；
outputs：编译输出临时产物，未打包的资源、编译后 js/arkts 字节码；
report：构建日志目录，记录编译报错、警告、构建耗时、打包信息，编译异常时可排查问题。

3.2 .idea IDE 配置目录
DevEco Studio 编辑器本地工作环境配置，仅本机生效，不会影响虚拟机运行逻辑，.gitignore 默认忽略该目录。
.deveco：鸿蒙专属 IDE 缓存，包含虚拟机设备缓存、模块识别缓存文件 project.cache.json；
modules.xml：记录项目内所有业务模块（本项目仅 entry 一个模块）；
workspace.xml：IDE 工作空间配置，记录打开页面、字体、主题、编译运行参数。

3.3 .gitignore Git 忽略配置文件
用于定义不需要上传至 GitHub 远程仓库的文件 / 目录，过滤编译缓存、本地 IDE 配置、虚拟机缓存、打包产物，避免仓库冗余文件，核心忽略内容：
编译目录 .hvigor/、entry/build/；
IDE 配置目录 .idea/；
第三方依赖目录 oh_modules/；
虚拟机本地缓存、日志、临时打包文件。

3.4 hvigor 全局构建配置文件夹
内含 hvigor-config.json5，是整个项目的构建环境总配置：
定义 hvigor 工具版本、编译插件加载规则；
统一配置全局编译参数、代码校验规则、混淆开关；
控制全项目统一 API 版本、兼容最低系统版本。

3.5 根目录核心构建脚本文件
3.5.1 build-profile.json5（项目全局构建总配置）
项目顶层编译配置，管控全部模块打包规则：
设置项目支持的编译模式：debug 调试模式、release 发布模式；
统一配置全项目 API 最低兼容版本；
管理多模块打包输出路径、签名打包配置、虚拟机调试参数；
定义编译时资源压缩、代码压缩规则。
3.5.2 hvigorfile.ts（项目根构建脚本）
基于 TS 的自定义构建任务脚本，可拓展自定义打包、拷贝资源、代码校验、自动化脚本，本项目默认使用内置构建逻辑，无自定义任务。
3.5.3 oh-package.json5（全局依赖管理文件）
OpenHarmony 项目依赖管理核心文件，作用等同于前端 package.json：
定义项目名称、版本号、开源协议、描述；
dependencies 节点记录项目引入的第三方开源 ArkTS 工具库；
devDependencies 开发阶段依赖：单元测试、代码格式化、混淆工具；
声明共享依赖包，供 entry 模块直接引用。

3.6 辅助文档与资源目录
screenshots：存放应用运行截图，用于 README 文档展示效果；
LICENSE：开源协议声明，规定代码复制、修改、商用限制；
README.md / README.en.md：中英文项目介绍，包含工程下载、运行步骤、基础功能说明；
External Libraries：IDE 自动加载的系统 API 依赖库，包含 UI 组件、页面路由、窗口、数据存储系统接口。

4 AppScope 全局应用作用域深度解析
AppScope 是 Stage 模型多模块共享层，项目内所有业务模块均可读取该目录资源与配置，独立于 entry 模块，优先级高于模块私有资源。

4.1 resources 全局资源目录
存放全应用通用静态资源，entry 页面可直接引用，无需在模块内重复定义：
base/media：全局共用图片、图标、背景图，如应用桌面图标；
base/element：全局统一文字、颜色、尺寸常量，统一 APP 基础配色、字体大小；
多语言目录 zh_CN / en_US：全局中英文文字资源，切换系统语言自动适配。

4.2 app.json5 应用顶层全局配置文件
控制应用整体属性，系统安装、启动时优先读取：
app 节点：应用包名、应用名称、版本号、版本名称；
bundleType：应用类型为普通应用 app；
vendor：开发厂商信息；
requestPermissions：应用全局所需系统权限（本计算器无需特殊权限，仅基础窗口权限）；
window：全局窗口默认宽高、适配屏幕、默认横竖屏模式。

5 entry 业务入口模块（计算器核心源码，最关键层级）
entry 是项目唯一可运行模块，计算器所有 UI 页面、四则运算逻辑、按钮交互、模块私有资源全部存放在此目录。
5.1 build 模块编译输出目录
编译自动生成，存放 entry 模块编译产物：资源编译文件、字节码、未签名安装包，每次 clean 工程可清空，无需提交仓库。

5.2 entry 模块配置文件
5.2.1 build-profile.json5
仅作用于 entry 模块的编译配置，独立于全局配置：
设置 entry 模块 API 版本、编译目标设备；
单独配置模块 debug/release 打包参数；
配置模块资源压缩、图片压缩规则。
5.2.2 hvigorfile.ts 模块级构建脚本
仅针对 entry 模块生效，定义模块打包、资源拷贝、代码校验任务。
5.2.3 oh-package.json5 entry 模块依赖清单
单独声明当前计算器模块需要引入的第三方工具库，与根目录依赖区分，仅当前模块生效。

5.3.1 EntryAbility.ets 应用生命周期入口
程序启动第一个执行的文件，管控 APP 完整生命周期：
onCreate()：应用创建时触发，初始化全局状态、窗口配置；
onForeground()：应用切到前台可见时触发；
onBackground()：应用退到后台隐藏时触发；
onDestroy()：应用彻底关闭销毁资源；
程序运行逻辑：系统启动 APP → 执行 EntryAbility → 读取 module.json5 路由 → 跳转 Index.ets 计算器首页。
5.3.2 pages/Index.ets 计算器主页面
整个项目业务逻辑核心，分为两大部分：UI 布局 + 四则运算交互逻辑
UI 布局部分：
顶部文本框：展示输入表达式与计算结果；
网格布局按钮：数字 0-9、加减乘除运算符、清除键、小数点、等于键；
样式绑定 resources 内颜色、尺寸资源。
业务运算逻辑：
@State 响应式变量缓存用户输入字符串、运算结果；
数字按钮点击事件：拼接数字到表达式；
运算符点击事件：缓存运算符、分割左右运算数；
等于按钮点击：解析表达式，执行四则运算，捕获除零异常；
清除按钮：重置输入、清空结果面板。
5.3.3 resources 模块私有资源
仅计算器首页可使用，优先级高于全局 AppScope 资源：
media：计算器界面背景、按钮图标；
element：按钮文字颜色、面板背景色、字体大小、圆角尺寸；
多语言文件夹：中英文切换时自动替换按钮文字。
5.3.4 profile/main_pages.json 页面路由配置
注册当前模块所有页面路径，系统根据该文件找到默认启动页面：
json
{
  "src": [
    "pages/Index/Index"
  ],
  "window": {
    "designWidth": 720,
    "autoDesignWidth": true
  }
}
src 数组内页面为模块可访问页面，数组第一个页面为程序默认启动页。
5.3.5 module.json5 entry 模块核心配置文件
entry 模块最关键配置，系统启动页面依赖此文件：
pages 节点：关联 main_pages.json 路由文件；
abilities 节点：声明 EntryAbility 应用入口；
window 窗口配置：计算器窗口尺寸、适配折叠屏、默认竖屏；
requestPermissions：模块所需权限；
metadata：桌面图标、应用名称绑定 AppScope 资源。

6 计算器原有基础功能完整执行流程
虚拟机启动应用，系统读取 AppScope/app.json5 获取应用全局信息；
加载 entry 模块 module.json5，识别应用入口 EntryAbility.ets；
执行 EntryAbility.onCreate () 完成应用初始化；
根据 main_pages.json 路由配置，自动跳转 Index.ets 计算器首页；
Index.ets 加载 entry/resources 资源渲染全部按钮、输入面板 UI；
用户点击数字 / 运算符按钮，@State 变量实时更新界面表达式；
点击等于按钮，解析表达式字符串，完成四则运算并渲染结果；
点击清除键，清空输入表达式与计算结果；
应用切后台 / 关闭，触发 EntryAbility.onDestroy () 销毁页面资源，所有运算记录仅内存存储，退出后丢失。

7 现有工程优缺点分析 & 拓展功能铺垫（任务 3 新增功能依据）
7.1 工程优点
架构规范：遵循鸿蒙 Stage 标准分层，全局资源与模块资源分离，配置文件职责清晰；
低耦合：页面 UI 布局与运算逻辑写在同一页面但可拆分，便于后续抽离工具类；
轻量化：无冗余第三方依赖，编译速度快，虚拟机运行流畅；
易拓展：基础四则运算逻辑独立，可直接新增历史记录、正负转换、精度优化等功能。
